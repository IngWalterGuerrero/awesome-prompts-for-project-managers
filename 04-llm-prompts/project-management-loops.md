# Project Management Loops

## Category

llm-prompts

## Recommended Use

Design repeatable project management loops for reviews, escalation, coordination, and continuous improvement.

## Ideal For

- Weekly project reviews
- PMO operating rhythms
- Risk and issue loops
- Lessons learned

## Compatible Tools

- ChatGPT
- Claude
- Gemini

## Level

advanced

## Variables

- `[PROJECT_TYPE]`
- `[TEAM_STRUCTURE]`
- `[CADENCE]`
- `[MAIN_PROBLEMS]`
- `[OUTPUT_FORMAT]`

## Prompt

```text
Design a project management operating loop for a [PROJECT_TYPE]. Team structure: [TEAM_STRUCTURE]. Cadence: [CADENCE]. Main problems: [MAIN_PROBLEMS]. Output format: [OUTPUT_FORMAT]. Define the loop objective, inputs, meeting rhythm, decision points, escalation rules, metrics, owner roles, and follow-up actions. Make it practical enough to implement next week.
```

## Example Use

```text
Design a project management operating loop for a delayed substation project with engineering, procurement, construction, and client stakeholders.
```

## Expected Output

An actionable PM operating loop with cadence, responsibilities, metrics, and escalation logic.

## Notes

Ask the model to produce a RACI table if ownership is unclear.
